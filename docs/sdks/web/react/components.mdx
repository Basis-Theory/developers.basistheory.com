# Elements Components

## CardElement

The `CardElement` [component](https://reactjs.org/docs/components-and-props.html) features a full credit card form, wrapping the [Card Element type](/docs/sdks/web/web-elements/types#card-element) functionality and taking care of the React lifecycle. The Card Element contains the following inputs:

- _cardNumber_
- _expirationDate_
  - Must not be expired or greater than 19 years in the future
- _cvc_

<BasisTheoryComponentDisplay title="Live Card Element">
  <CardElement id="types-card-element" />
</BasisTheoryComponentDisplay>

```javascript showLineNumbers
import {
  BasisTheoryProvider,
  CardElement,
  useBasisTheory,
} from "@basis-theory/basis-theory-react";

const App = () => {
  // creates a new instance of BasisTheory class
  const { bt } = useBasisTheory("<API_KEY>", { elements: true });

  return (
    <BasisTheoryProvider bt={bt}>
      <MyComponent />
    </BasisTheoryProvider>
  );
};

const MyComponent = () => {
  return <CardElement id="myCard" />; // Element will grab instance from the Context
};
```

## TextElement

## Error Handling

Any errors occurring during the element mounting phase are thrown intentionally during the render cycle. Consider using React's [error boundaries](https://reactjs.org/docs/error-boundaries.html) to handle these errors gracefully. When using React class components, you can also implement the methods [`getDerivedStateFromError`](https://reactjs.org/docs/react-component.html#static-getderivedstatefromerror) and [`componentDidCatch`](https://reactjs.org/docs/react-component.html#componentdidcatch) to handle component errors appropriately.
