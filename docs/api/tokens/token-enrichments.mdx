# Enrichments

Basis Theory offers several features that can be used to enrich the tokens that are tokenized.

| Attribute     | Type  | Token Types                                                                                    | Description                                                        |
| ------------- | ----- | ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| `bin_details` | _any_ | [`card`](docs/api/tokens/token-types#card) , [`card_number`](docs/api/tokens/token-types#card) | Object containing information related to the tokenized credit card |

## BIN Details <Enterprise />

BIN details, or Bank Identification Number details, represent a fundamental aspect of credit card processing.
Each credit card is associated with a unique BIN, which serves as a numerical identifier for the issuing bank
or financial institution. These details hold crucial technical significance as they help validate transactions,
prevent fraud, and optimize routing. Understanding the intricate mechanics of BIN details
is vital for implementing secure and efficient credit card processing solutions.

In this section of Basis Theory's documentation, we outline the options you have when interacting with
BIN details within the Basis Theory platform and how our tokenization services
seamlessly integrate this crucial element to enhance data security and compliance in your applications.

Basis Theory offers two options for BIN details:

| Level                      | Description                                                                                  |
| -------------------------- | -------------------------------------------------------------------------------------------- |
| [Standard](#standard-data) | Details on the credit cards function and origin.                                             |
| [Enhanced](#enhanced-data) | Include Standard Data and also interchange, authentication, and other more advanced details. |
